## 4-2. 데이터 타입과 데이터 변환(CAST, SAFE_CAST)

~~~
✅ 학습 목표 :
* 데이터 타입의 종류를 설명할 수 있다. 
* 데이터 타입을 변환하는 방법을 설명할 수 있다. 
~~~

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
데이터 타입은 숫자, 문자, 시간, 날짜 부울 등등
- 숫자: 정수 1, 2, 3, 4,
- 문자: “나”, “너”
- 시간, 날짜: 2024-01-01
- 부울: 참/거짓

데이터 타입이 중요한 이유
- 보이는 것과 저장된 것의 차이가 존재
- 숫자 혹은 문자의 구별
- 빈 값 혹은 NULL

자료 타입 변경
- 함수: CAST
SELECT
CAST(1 AS STRING)  
#숫자 1을 문자 1로 변경

SAFE_CAST는 변환이 가능하면 값을 반환하고, 불가능하면 오류 대신 NULL을 반환

수학 함수
- 수학 연산(평균, 표준편차, 코사인 등)이 존재\

## 4-4. 날짜 및 시간 데이터 이해하기(1) (타임존, UTC, Millisecond, TIMESTAMP/DATETIME)

~~~
✅ 학습 목표 :
* 날짜 및 시간 데이터 타입과 UTC의 개념을 설명할 수 있다. 
* DATE, DATETIME, TIMESTAMP 에 대해서 설명할 수 있다.
* 시간함수들의 종류와 시간의 차이를 추출하는 방법을 설명할 수 있다. 
~~~

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
날짜 및 시간 데이터의 핵심
- 유저가 특정 행위를 하면 반드시 시간이 붙는다
- 일반적인 시간과, 개발할 때의 시간 개념이 다르다
1) 날짜 및 시간 데이터 타입 : DATE, DATETIME, TIMESTAMP
2) 날짜 및 시간 데이터 관련 알면 좋은 내용: UTC, Millisecond
3) 날짜 및 시간 데이터 타입 변환하기
4) 시간 함수(두 시간의 차이, 특정 부분 추출하기)

DATE: 날짜만 표시 2023-10-31
DATETIME: DATE+시간
TIME: 시간만

타임존
GMT: 
- 영국의 그리니치 천문대 기준으로 시간 조율 
- 한국시간: GMT +9

UTC: 
- 국제적인 표준 시간
- 타임존이 존재한다: 특정 지역의 표준 시간대

TIMESTAMP
- 시간 도장
- UTC로부터 경과한 시간
- TIME Zone 정보가 있다

Millisecond
- 천분의 1초
- 빠른 반응이 필요한 분야에서 사용
- 정밀한 데이터
- Millisecond->timestamp->datetime

microsecond
- 백만분의 1초

=> datetime과 timestamp의 타임존을 절대 누락하지 않기

Extract
- 2024-01-02 14:00:00
- 위 시간에서 특정 월이나 일을 추출하고자 할 때
- 요일 추출: DAYOFWEEK
- 첫날이 일요일인 1~7까지의 값

DATETIME_TRUNC
- 2024-01-02 14:41:21
- 함수 사용하면 2024-01-02 14:00:00

PARSE_DATETIME
- 문자열로 저장된 DATETIME을 DATETIME 타입으로 바꿀 때
- 파싱하다

FORMAT_DATETIME
- DATETIME 타입으로 저장된 DATETIME을 문자열 데이터로 바꿀 때

DATETIME_DIFF
- 두 DATETIME의 차이를 알고 싶은 경우

## 문제 1

> **🧚Q. 프로그래머스 문제를 풀던 규서는 여러 번의 시행착오 끝에 결국 혼자 해결하기 어려워 오류 메시지를 공유하며 도움을 요청했습니다. 여러분들이 오류 메시지를 확인하고, 해당 SQL 쿼리에서 어떤 부분이 잘못되었는지 오류 메시지를 해석하고 찾아 설명해주세요.**

~~~sql
# 조건에 맞는 도서 리스트 출력하기
# 규서의 SQL 첫 번째 풀이
SELECT BOOK_ID, PUBLISHED_DATE
FROM BOOK
WHERE CATEGORY = '인문'
  AND YEAR(PUBLISHED_DATE, 2021);
  
오류 메시지 : Error: Number of arguments does not match for function YEAR
~~~



~~~
메세지는 함수 YEAR이 잘못되었음을 나타내며, YEAR(PUBLISHED_DATE)= 2021); 로 수정해야 한다
~~~
