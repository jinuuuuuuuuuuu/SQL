# 1️⃣ 개념정리 
<!-- 강의 수강 이후에 아래의 학습 목표에 맞게 개념을 자유롭게 정리해주세요.-->
## 1-1. BigQuery 기본지식

~~~
✅ 학습 목표 :
* 데이터 관련 기초 지식(OLTP, SQL, Row, Column, 저장 형태 등)을 설명할 수 있다. 
* BigQuery 관련 기초 지식에 대해서 파악할 수 있다. 
~~~

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
데이터는 보통 데이터베이스과 테이블 등에 저장, 이때 테이블은 데이터베이스 안에 포함
1) OLTP
- 거래에 주로 사용되는 데이터베이스 (음식 주문)
- 보류나 중간 상태가 없다(주문을 완료하거나 안하거나)
- 데이터의 추가와 데이터의 변경이 많이 발생한다(주문 업데이트)
- 분석 능력이 부족할 때는 OLAP를 사용
- OLAP는 데이터웨어하우스(데이터 창고)와 연결
- BigQuery는 구글 클라우드의 OLAP+데이터웨어하우스

2) SQL
- 데이터베이스에서 데이터를 가지고 올 때 사용되는 언어
- 퀴리문, 퀴리를 짠다 등등으로 표현

3) BigQuery
- SQL를 사용해 쉬운 데이터 추출
- OLAP 도구로 속도가 빠름
- 앱과 웹의 데이터를 쉽게 추출
- 클라우드 기반이어서 서버 필요 없음

- BigQuery 비용
- 쿼리 비용은 SQL할 때의 비용
- 저장 비용은 저장할 때의 비용

4) 테이블에 저장되는 형태
- 행은 로우, 열은 컬럼
- 로우는 하나의 고유 데이터를 의미(김진우, 김진오 등) (raw 데이터와 구분)
- 컬럼은 각 데이터의 특정 속성값(나이, 거주지 등)

5) BigQuery
- 프로젝트>데이터셋>테이블


## 2-1. 데이터 활용 Overview

~~~
✅ 학습 목표 :
* 데이터를 활용하는 과정을 설명할 수 있다.
* 데이터를 탐색하는 과정으로 조건과 추출, 요약을 할 수 있다. 
~~~

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
1) 문제정의
- MECE하다
2) 원하는 것 정하기
3) 데이터 탐색
- 단일자료와 다량의 자료로 나뉨
- 조건, 추출, 변환, 요약의 과정을 거친다
4) 데이터 결과 검증
- 예상과 실제가 같은지
5) 피드백/활용

- SQL은 데이터 탐색과 데이터 결과 검증 파트에서 사용된다


## 2-2. 저장된 데이터 활용하기

~~~
✅ 학습 목표 :
* 데이터가 저장되는 형태를 알고 저장된 데이터를 활용할 수 있다. 
~~~

<!-- 새롭게 배운 내용을 자유롭게 정리해주세요.-->
쿼리작성 전 생각하기
- 어떤 데이터가 어떻게 저장되어 있는지 생각하기
- 컬럼의 의미는 무엇인지
- 데이터를 추출하기 전에 데이터웨어하우스에 데이터가 어떻게 저장되어 있는지를 확인

데이터가 저장되는 형태: ERD
- 데이터베이스 구조를 한 눈에 알아보기 위해 사용
- 테이블에 여러 컬럼들이 들어있다
- 각 테이블의 컬럼들이 연결되어 있기도 하다(order와 customer 테이블이 customer_id 컬럼을 공유)
- 어떤 테이블과 컬럼이 있는지 확인
- 다른 테이블과 연결할 때 어떤 컬럼을 사용하는지 확인

예시: 이커머스 산업의 데이터 
- 상품
- 유저
- 주문
- 장바구니에 담은 물건
- 웹페이지 접근 수
